What is JSDoc?

It's a way to add type safety to pure JavaScript files

There are times when, from a consumer TypeScript file, we want to consume files that are in pure JavaScript and are too large to convert to a TypeScript file

In those instances we just want to add type safety to those parts of that JavaScript file so that our consumer TypeScript file can carry on with its own type safety

We can do this by adding type annotations with JSDoc comments in the consumee pure JavaScript file

This means that we have spectrum of type safety when it comes to JavaScript and TypeScript

No way to specify optionality on params before ES2015, all function params are considered optional; however, supplying too many arguments results in an error as that _can_ be checked by TypeScript
